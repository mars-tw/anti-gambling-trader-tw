# 方法論：我們如何分辨「優勢」與「賭博」

這份文件說明本工具背後的統計推理，讓你能信任（並質疑）它的結論。

## 核心問題

> 一個人交易一段時間後賺了錢。這代表他有本事，還是只是運氣好？

這不是哲學問題，而是一個**可以用統計檢定回答**的問題。

## 第一道關卡：期望值（Expectancy）

每筆交易的期望損益：

```
期望值 = 勝率 × 平均獲利 − 敗率 × 平均虧損
```

**這是最重要的單一數字。** 期望值為負，代表交易越多次、賠越多，
數學上注定長期虧損 —— 不論短期帳面多漂亮。

⚠️ 常見陷阱：
- **高勝率 ≠ 賺錢**。勝率 90% 但「賺小賠大」，一次大虧就回吐所有獲利。
- **高盈虧比 ≠ 賺錢**。盈虧比 5，但勝率只有 13%，期望值仍可能是負的。

  （見 `examples/crypto_luck.json`：盈虧比 4.32，期望值卻是 −192。
  因為總獲利的 92.9% 來自單筆迷因幣暴賺 —— 那是中樂透，不是方法。）

## 第二道關卡：這個正期望，會不會只是運氣？

就算期望值為正，樣本小的時候，它也可能只是隨機波動。我們用兩種互補的檢定：

### t 檢定

假設 H₀：真實平均損益 ≤ 0。算出在常態近似下，觀察到目前這麼好的
平均值「純屬偶然」的機率（p 值）。p < 0.05 才拒絕「這是運氣」。

對小樣本（df < 30）我們做了**保守校正**，刻意讓 p 值偏大 ——
寧可少判定為顯著，也不要把運氣誤認為實力。

### Bootstrap 重抽樣

不假設任何分布（交易損益通常嚴重偏態、厚尾，常態假設未必成立）。
做法：從你的交易裡「有放回地」重抽出同樣多筆，算平均；重複幾千次，
看有多少比例的重抽平均值 ≤ 0。這個比例就是經驗 p 值。

**只有 t 檢定與 bootstrap 雙雙 p < 0.05，我們才認定「正期望值顯著」。**

## 第三道關卡：樣本外驗證（揭穿過度配適與倖存者偏差）

一個策略「在歷史資料上很賺」，可能只是因為它剛好貼合了那段歷史。
要戳破這種假象：

1. 把交易**依時間**切成前段（樣本內 70%）與後段（樣本外 30%）。
2. 看前段展現的正期望，在後段是否仍然存在、衰退多少。

如果優勢在樣本外**消失或翻負**，那它八成是雜訊或過度配適 ——
也就是說，過去的獲利更可能是運氣，而非能延續到未來的真本事。

> **倖存者偏差**：你只看到「自己這次賺了」，看不到「無數用同樣方法
> 賠光退場的人」。樣本外驗證，就是在你自己的資料裡模擬「未來」，
> 看你的方法會不會也成為那批退場者之一。

## 第四道關卡：賭博特徵掃描

即使前三關都過，我們仍會掃描結構性風險警訊：

| 警訊 | 為什麼危險 |
|------|-----------|
| 負期望值 | 數學上注定虧損 |
| 獲利過度集中單筆 | 靠運氣中一次，非可重複方法 |
| 高勝率 + 極差盈虧比 | 賺小賠大，一次爆倉吃光獲利 |
| 極端回撤（>50%） | 過程曾瀕臨毀滅，多數人撐不過 |
| 長連虧（≥8 次） | 心理上極難守紀律 |
| 純當沖且不顯著 | 最接近賭場的型態，成本與雜訊極高 |
| 獲利因子接近 1 | 毫無安全邊際，扣成本後很可能在虧 |

## 為什麼門檻偏保守？

> 寧可錯殺（把賭博誤判為賭博），不可放過（把賭博說成優勢）。

- 前者的代價：使用者多存疑、多驗證，浪費一些時間。
- 後者的代價：使用者誤信自己有本事而重押，可能賠上畢生積蓄。

兩種錯誤的代價天差地遠，所以我們刻意偏向保守。

## 限制與誠實聲明

- 「具統計優勢」是**目前為止**的證據，**不是未來的保證**。市場會變，優勢會衰減。
- 成本估算用各市場常見預設值，可能與你的實際費率有出入（請自行調整）。
- 樣本外驗證用的是「使用者交易紀錄的時間穩定度」，不是用外部行情重跑策略 ——
  它驗證的是「你過去這套執行的穩定性」，而非「策略在任意市場的普適性」。
- 統計顯著不等於經濟上有意義：通過檢定的微小優勢，可能不值得承擔交易的壓力。
